# Участники группы 251-321 и личный вклад в проект

> Данный документ описывает трёх студентов группы **251-321**, присоединившихся к фронтенд-отделу проекта «Сайт службы психологической помощи Московского Политеха» в 2026 году в рамках дисциплины «Проектная деятельность». Мы — небольшая часть большой команды (20+ человек, 10+ отделов).
>
> Фронтенд проекта построен по методологии **Feature-Sliced Design (FSD)**. Ниже указано, в каких слоях архитектуры находятся наши компоненты.

---

## Жемчугов Алексей

**Роль:** Frontend · Header & News

### Выполненные задачи

- **Navbar (слой `widgets`):** React-компонент шапки с логотипом, навигационным меню через React Router, адаптивным бургером с CSS-анимацией и управлением фокусом (keyboard accessibility).
- **Страница новостей (слои `pages`, `entities`, `features`):** карточки на TypeScript-интерфейсах, серверная пагинация и фильтрация по тегам через TanStack Query.
- **Конфигурация проекта:** настройка Vite, путей TypeScript, ESLint, Prettier и `.editorconfig` для единого стиля кода во всей команде.
- **Личный кабинет психолога (совместно):** список заявок (Appointments), карточка клиента.

### Использованные технологии

React 18, TypeScript, React Router DOM, TanStack Query, Vite, ESLint, Prettier, SCSS-модули

---

## Зарубин Дмитрий

**Роль:** Frontend · Adaptive & Modal

### Выполненные задачи

- **Адаптивность компонентов:** корректное отображение всех React-компонентов для 5 брейкпоинтов — от 320 px (мобильные) до широких десктопных экранов (2560 px) через SCSS-миксины и media queries.
- **Модальное окно записи на приём (слой `features`):** React-компонент с React Hook Form + Zod для валидации, управлением фокусом (focus trap) и закрытием по клавише Escape.
- **CSS-анимации и переходы:** плавные анимации открытия/закрытия и оверлеи для интерактивных компонентов через SCSS.
- **Личный кабинет психолога (совместно):** форма обратной связи, статусные бейджи.

### Использованные технологии

React 18, TypeScript, React Hook Form, Zod, SCSS-модули, media queries

---

## Якунин Егор

**Роль:** Frontend · Dark Theme

### Выполненные задачи

- **Система тем (слой `shared`):** реализована на CSS Custom Properties без дублирования CSS-правил — переключение темы не перегружает браузер перерисовкой.
- **Переключатель тем (слой `features`):** хранение выбора пользователя через Zustand (синхронизация с localStorage), автоматическая поддержка системного `prefers-color-scheme`.
- **WCAG 2.1 AA:** обеспечено соответствие контраста всех текстовых элементов стандарту доступности в обеих темах (светлой и тёмной).
- **Личный кабинет психолога (совместно):** тёмное оформление таблицы расписания.

### Использованные технологии

React 18, TypeScript, CSS Custom Properties, SCSS-модули, Zustand, localStorage, `prefers-color-scheme`
