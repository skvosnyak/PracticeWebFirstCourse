# Описание проекта «Сайт службы психологической помощи Московского Политеха»

## Аннотация

Проект «Сайт службы психологической помощи Московского Политеха» — цифровая платформа для онлайн-записи студентов на психологические консультации. Разрабатывается с 2024 года большой межфункциональной командой (20+ участников, 10+ отделов). Мы трое — студенты группы 251-321 — присоединились к фронтенд-отделу в 2026 году в рамках дисциплины «Проектная деятельность».

Действующая платформа: [psychohelp-mospoly.ru](https://psychohelp-mospoly.ru/)

---

## Цель проекта

Создать удобный и безопасный онлайн-ресурс для записи студентов на психологические консультации и взаимодействия с психологами Московского Политеха без психологических и технических барьеров.

---

## Актуальность

По данным Всемирной организации здравоохранения, депрессия и тревожные расстройства входят в топ-3 причин академических неудач среди молодёжи. Внутреннее исследование Московского Политеха подтвердило острую потребность в доступной психологической поддержке: телефонная запись отпугивает студентов — им нужен анонимный цифровой канал связи с психологом.

---

## Задачи разработки

1. Разработать систему регистрации и аутентификации с JWT-токенами и разделением ролей (студент, психолог, администратор).
2. Реализовать механизм подачи заявок на консультацию с выбором формата (очно / онлайн) и удобного времени.
3. Создать личный кабинет студента с историей обращений, текущими статусами заявок и возможностью отмены.
4. Создать личный кабинет психолога с расписанием, очередью заявок и инструментами обратной связи со студентом.
5. Разработать административный интерфейс для управления расписанием, назначения психологов и просмотра статистики.
6. Реализовать систему email-уведомлений о каждой смене статуса заявки.
7. Обеспечить полную адаптивность интерфейса для экранов от 320 до 2560 пикселей.
8. Внедрить тёмную тему с сохранением предпочтений пользователя и поддержкой системного `prefers-color-scheme`.
9. Обеспечить доступность интерфейса по стандарту WCAG 2.1 AA: навигация с клавиатуры, достаточный контраст, aria-атрибуты.

## Техническая архитектура

### Frontend

**Стек:** React 18 + TypeScript, сборка через Vite.

**Архитектура:** [Feature-Sliced Design (FSD)](https://feature-sliced.design/) — методология разделения кода на независимые слои.

| Слой       | Назначение                                                            |
| ---------- | --------------------------------------------------------------------- |
| `app`      | Инициализация, маршруты, провайдеры                                   |
| `pages`    | Страницы приложения по URL-маршрутам                                  |
| `widgets`  | Крупные переиспользуемые UI-блоки (например, Navbar)                  |
| `features` | Функциональность с бизнес-ценностью (например, модальное окно записи) |
| `entities` | Типы, API-запросы, логика сущностей (Appointments, Reviews)           |
| `shared`   | UI-кит, утилиты, хелперы, CSS-переменные, константы                   |
| `tests`    | Конфигурация и утилиты для тестирования                               |

**Управление состоянием:** TanStack Query — серверное состояние и кэширование; Zustand — глобальное клиентское состояние (тема, сессия). Формы — React Hook Form + Zod. HTTP-запросы — Axios.

**Стилизация:** SCSS-модули (22,8% кодовой базы), CSS Custom Properties для системы тем.

**Тесты:** Vitest — юнит- и компонентные тесты; Playwright — e2e-тесты.

**Качество кода:** ESLint, Prettier, Husky + lint-staged (проверка при коммите).

**Мок-сервер:** встроенный `mok-server` — запуск фронтенда без реального бэкенда.

### Backend

**Стек:** Python + FastAPI + Uvicorn (ASGI-сервер). Асинхронный драйвер AsyncPG для PostgreSQL. UV — менеджер пакетов.

**Архитектура** — многоуровневая:

| Слой           | Назначение                             |
| -------------- | -------------------------------------- |
| `models`       | SQLAlchemy-модели, схема базы данных   |
| `repositories` | Слой доступа к данным                  |
| `services`     | Бизнес-логика приложения               |
| `controllers`  | Обработка API-запросов (FastAPI-роуты) |
| `schemas`      | Валидация данных через Pydantic        |

**Основные сущности:** Users, Roles, Therapists (психологи), Appointments (записи), Reviews (отзывы).

**Безопасность:** JWT-аутентификация, Passlib для хэширования паролей, миграции через Alembic.

**Тесты:** Pytest с поддержкой асинхронного кода, покрытие через pytest-cov.

### База данных

PostgreSQL + ORM SQLAlchemy. Управление миграциями через Alembic. Индексы на часто запрашиваемых полях.

### Инфраструктура

Docker + Docker Compose (три контейнера: FastAPI, PostgreSQL, Nginx). Nginx — reverse proxy и раздача статики. CI/CD через GitHub Actions.

## Жизненный цикл заявки (Appointment)

```
Новая → Рассматривается → Подтверждена → Завершена
                                        ↘ Отменена
```

На каждом переходе студенту автоматически отправляется email-уведомление, а в личном кабинете обновляется статус в режиме реального времени.

---

## Конфиденциальность

Данные консультаций хранятся в зашифрованном виде. Доступ к информации о клиентах имеет только назначенный психолог. Платформа соответствует требованиям ФЗ-152 о персональных данных.
