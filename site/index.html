О проекте

Цели, задачи и актуальность

Платформа, которая делает психологическую помощь доступной для каждого студента Московского Политеха.

Важно: вы читаете учебный отчёт в формате сайта, подготовленный тремя студентами группы 251-321 в рамках дисциплины «Проектная деятельность». Здесь описан проект, в котором мы участвуем, и наш конкретный вклад. Это не сам сайт Службы Психологической Помощи — он доступен по адресу psychohelp-mospoly.ru.

Проект «СПП Московского Политеха» существует уже более двух лет и ведётся большой межфункциональной командой. По данным Всемирной организации здравоохранения, депрессия и тревожные расстройства входят в топ-3 причин академических неудач среди молодёжи. Именно поэтому в Московском Политехе решили создать цифровую платформу — чтобы студентам было проще записаться к психологу, минуя звонок или личный визит.

Цель проекта

Создать удобный и безопасный онлайн-ресурс для записи студентов на психологические консультации и взаимодействия с психологами Московского Политеха без психологических и технических барьеров.

Актуальность

Данные ВОЗ и внутреннее исследование университета указывают на острую потребность в доступной психологической поддержке. Телефонная запись отпугивает студентов — им нужен анонимный цифровой канал связи с психологом.

Задачи разработки

Реализовать систему ролей и аутентификации, личные кабинеты студента и психолога, механизм подачи и обработки заявок, уведомления по email, адаптивный дизайн и тёмную тему.

Конфиденциальность

Данные консультаций хранятся в зашифрованном виде. Доступ к информации о клиентах имеет только назначенный психолог. Платформа соответствует требованиям ФЗ-152 о персональных данных.

Конкретные задачи проекта

  • Разработать систему регистрации и аутентификации с JWT-токенами и разделением ролей (студент, психолог, администратор)
  • Реализовать механизм подачи заявок на консультацию с выбором формата (очно / онлайн) и удобного времени
  • Создать личный кабинет студента с историей обращений, текущими статусами заявок и возможностью отмены
  • Создать личный кабинет психолога с расписанием, очередью заявок и инструментами обратной связи со студентом
  • Разработать административный интерфейс для управления расписанием, назначения психологов и просмотра статистики
  • Реализовать систему email-уведомлений о каждой смене статуса заявки
  • Обеспечить полную адаптивность интерфейса для экранов от 320 до 2560 пикселей на всех устройствах
  • Внедрить тёмную тему с сохранением предпочтений пользователя в localStorage и поддержкой системного prefers-color-scheme
  • Обеспечить доступность интерфейса по стандарту WCAG 2.1 AA: навигация с клавиатуры, достаточный контраст, aria-атрибуты

Техническая архитектура

Frontend

HTML5, CSS3, JavaScript без фреймворков. Чистый, быстрый интерфейс с нулевыми сторонними зависимостями в браузере. Взаимодействие с сервером через Fetch API.

Backend

Python + FastAPI + Uvicorn (ASGI-сервер). AsyncPG — асинхронный драйвер PostgreSQL. Pydantic — валидация данных, Passlib — хэширование паролей, JWT — аутентификация. Пакеты управляются через UV.

База данных

PostgreSQL с ORM SQLAlchemy. Управление миграциями через Alembic. Индексы на часто запрашиваемых полях: student_id, status, created_at.

Инфраструктура

Docker + Docker Compose для контейнеризации и воспроизводимого окружения. Nginx в роли reverse proxy и раздачи статики. CI/CD через GitHub Actions.

05 мар 2025
История проекта
Бэкенд: первый стабильный деплой платформы на тестовый сервер

Бэкенд-команда успешно задеплоила рабочую версию API на тестовый сервер. Docker Compose объединяет три контейнера: FastAPI-приложение, PostgreSQL и Nginx в роли reverse proxy. Все endpoint'ы аутентификации и заявок доступны по HTTPS.

Фронтенд получил стабильный адрес тестового сервера и начал полноценную интеграцию — отправку реальных запросов вместо заглушек. Бэкенд-команда подготовила документацию по API в Swagger UI (встроен в FastAPI).

28 фев 2025
История проекта
QA-отдел: настроено автоматизированное тестирование бэкенда через Pytest

Команда тестирования настроила инфраструктуру для автоматизированных тестов на Pytest. Покрыты ключевые маршруты: регистрация, вход, создание заявки, смена статуса, получение списка заявок. Тесты запускаются на изолированной тестовой базе данных.

В GitHub Actions добавлен шаг автозапуска тестов при каждом pull request. Это позволило команде уверенно принимать изменения без риска регрессий — особенно важно при активной разработке как на фронтенде, так и на бэкенде.

10 фев 2025
История проекта
Спроектирована архитектура бэкенда и схема жизненного цикла заявок

Бэкенд-отдел утвердил схему жизненного цикла заявки: «Новая» → «Рассматривается» → «Подтверждена» → «Завершена» / «Отменена». На каждом переходе генерируется email-уведомление студенту. Описаны все эндпоинты REST API: создание, получение, обновление статуса заявки.

Фронтенд-отдел получил техническое задание на реализацию соответствующих UI-компонентов: статусных бейджей, кнопок действий и стейт-машины на клиенте. Согласован формат JWT-токенов и схемы запросов/ответов.

14 янв 2025
История проекта
Бэкенд: завершена разработка ядра REST API — аутентификация, заявки, уведомления

Бэкенд-команда завершила реализацию основных endpoint'ов API: регистрация и вход (JWT), создание, получение и обновление статуса заявки, получение расписания психолога. Все схемы запросов и ответов описаны через Pydantic-модели и задокументированы в Swagger.

Настроена система email-уведомлений: при каждой смене статуса заявки студент получает письмо с подробностями. Подключён Alembic для управления миграциями базы данных. API готово к интеграции с фронтендом.

08 дек 2024
История проекта
Утверждён дизайн в Figma — передача макетов фронтенду

Дизайн-отдел завершил разработку полного комплекта макетов в Figma: главная страница, страница новостей, форма регистрации и входа, личный кабинет студента, личный кабинет психолога. Каждый макет представлен в двух вариантах — светлая и тёмная тема — и в трёх брейкпоинтах (мобильный, планшет, десктоп).

Фронтенд-отдел провёл разбор макетов совместно с дизайнерами, уточнил поведение интерактивных состояний (hover, focus, active, disabled) и согласовал финальные токены дизайн-системы. Работы переходят в фазу активной разработки.

20 ноя 2024
История проекта
Анализ аналогов и выбор технологического стека

Команда провела анализ существующих решений в области психологической поддержки студентов в российских университетах. Изучены платформы МГУ, СПбГУ и ряда зарубежных вузов. Выявленные сильные стороны аналогов легли в основу требований к платформе, слабые места — стали точками дифференциации.

По итогам технического совещания утверждён стек: Python + FastAPI на бэкенде, чистый HTML/CSS/JS на фронтенде, PostgreSQL в качестве СУБД, Docker для контейнеризации. Решение обусловлено опытом команды, требованиями к производительности и отсутствием лицензионных ограничений.
